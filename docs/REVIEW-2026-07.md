### 2. What deserves protection

The most valuable asset here is not a model — it is the **method**, and it is
rarer than any of the numbers:

- **Pre-registered bars, immutable once frozen; honest negatives as first-class
  results.** The ledgers close questions permanently (`RESEARCH-IDEAS.md` is
  mostly ~~struck-through~~ entries, each with its price).
- **Sacred artifacts.** `artifacts.lock.json` + `scripts/fetch_champions.py
  --check` make "which model flew this number" machine-checkable.
- **The instrument self-corrects.** Three caught-and-hardened cases are now
  house rules in `CLAUDE.md`: the probe split-identity leak (λ campaign), the
  blank-frame rendering bug that had mis-written two WM verdicts
  (`search_wm_v1/v2`, retracted by `search_wm_v3`), and the stale speed-1.0
  eval default that briefly faked an "N-room doesn't scale" result.
- **The cheap path, proven 4× in one release** (yaw, alt, height, person):
  frozen-latent probe → retrained head → choreography — **zero WM retrains in
  all of v0.8**. This is the pattern every new capability should enter through.
- **One knob per run; flight gates over model-axis AUC** (single-seed dense
  val AUC spreads 0.47–0.99 across same-draw seeds — measured, so nobody
  trusts that instrument again).

Any future assistant should read `CLAUDE.md` §"measured discipline" before
touching anything. It is the constitution; this review only adds furniture.

## Part II — the thesis: from glimpses to evidence

The spatial division of labor is settled and measured. Look at what remains
open, and every frontier is about **time**, not space:

- per-frame recall ~0.5–0.76 is compounded into mission recall ~0.93 by
  *choreography* (scan spins, confirm-2, approach-and-face) — ad-hoc temporal
  integration, never formalized (`vision_v1`, `yaw_v1`, `person_v1`);
- a doorway cannot be told from a box-pinch in one frame (AUC 0.631) — the
  named candidate is *trajectory-integrated* structure (`search_roomgraph_v1`);
- the dense transit frontier's residual deaths are FOV-edge side clutter,
  13/14 at ×2.0 — things that **slid out of view** (`experiments/dense_speedrun/`);
- stop-and-observe is OOD for continuous policies (`slalom_stopobserve_v1`);
- the dispatch arc's closing sentence: world identity needs *probing motion* —
  information that only exists across frames (`experiments/dispatch/`).

So the program's one scientific theme: **turn glimpses into evidence.** Rung
by rung: formalize decision-level evidence accumulation first (no retrain,
builds the judges), then — and only through an oracle probe and the owner's
gate — the one WM retrain of the horizon, memory in the model itself. The
theme compounds: the same move (single frame → integrated trajectory) applies
to the camera channel (C2) and the beam channel (C4); the temporal WM (C5), if
it survives its probe, feeds conditional calibration its missing context
signal; and the re-distillation machinery (K-series) is what makes any WM
improvement shippable without stranding the zoo again.

**On re-opening the GRU.** The v0.2 negative stands (it is in the bound-space
table). A temporal campaign is admissible only as a *re-opening with a
mechanism argument*, per the house rule (`RESEARCH-IDEAS.md`, chain-algorithm
entry). The argument, pre-written: (a) the v0.2 judge was single-seed
model-axis AUC — an instrument later measured to spread ~0.5 per seed; (b) the
failure signature now in hand (FOV-edge deaths, glimpse-limited recall) is
exactly out-of-frame information, which no non-recurrent stack can represent;
(c) the data scale on offer is 5–10× what the v0.2 GRU saw. C5 below is built
so that if privileged memory can't help, the retrain dies for free.

## Part III — the program

Three horizons, pure simulation. Every campaign follows the house shape:
Phase-0 probe → frozen bars → verdict (either way, a finding). Costs are
Mac-days; "background" means an overnight queue with `*-DONE` markers.

### Horizon 1 — 0–2 weeks: instruments before opinions

All zero-WM-risk, all parallel-safe, each yields a number or an instrument.

- **I1 — eval-default audit (0.5 d).** Hoist `INDOOR_SPEED = 0.6` + the
  deployable safety name into one constants module; make `eval_search.suite()`
  default to them; selftest-assert CLI and API defaults agree; print a config
  header in every suite. Then grep-audit every eval's defaults against its
  documented robust config.
- **G1 — head governance (1 d).** `artifacts.lock.json` v1.1, additive: lock
  the three *deployed* heads (`yaw`, `low`, `person`) with new optional fields
  `wm` (sha of the latent each head was trained on) and `track`;
  `fetch_champions` already tolerates extra keys (superset check). Extend
  `FlightMode` with head bindings + a `verify()` that cross-checks the lock;
  CI-safe consistency selftest (no artifacts needed). Refresh the release so a
  fresh clone restores a **complete flying system**. Owner decision folded in:
  canonicalize `alt` vs `alt_os` vs `low` (the eval path uses `low`).
- **Q1 — int8 parity probe (2–3 d; the missing centerpiece of writing/#8).**
  A ~150-line hand-rolled fake-quant wrapper (transparency over `torch.ao`):
  weights int8 per-channel for convs / per-tensor for linears, activations
  int8 per-tensor, bias int32 — matched to the GAP8 NNTool SQ8 flow; report a
  per-tensor-weights arm as the pessimistic bound. Calibrate on 512–2048
  frames from BOTH tracks (a unified WM calibrated on one distribution lies).
  Re-run the exact gates quantized: AUC@32 held-out, one closed-loop transit
  cell (n=60, same seeds), indoor detection AUCs, an n=30 indoor mission
  subset. **Bars frozen before any number**: ΔAUC@32 ≥ −0.010; Δcrash ≤ +3 pts
  and inside the float run's binomial CI; head ΔAUC ≥ −0.015. Pre-registered
  diagnostics: per-layer quantization SNR names the hostile layer; decision
  rules — hostile projection/pool → wider calibration; hostile predictor
  residual accumulation → int16 activations for the predictor only (flag as a
  bridge question); hostile heads → retrain heads on the quantized latent
  (seconds). Include a **grayscale-input arm** (the AI-deck's stock sensor is
  monochrome; the stack trains RGB) — prices that gap for pennies, inside the
  probe-level sim-to-real allowance. Honesty list (what sim cannot validate):
  PULP-NN rounding exactness, Autotiler tiling/DMA and real latency (the
  0.5 GMAC/s figure is explicitly challengeable), NNTool's actual per-layer
  choices, the real camera.
- **C1 — detect_inflight_v1 (1–2 d).** Pay the stationary-imaging debt:
  instrument the search runner to log per-frame head outputs + ground-truth
  visibility DURING flight at speed 0.6; replay-score. Pre-registered read:
  in-flight pooled AUC ≥ 0.90 = debt paid (an honest positive worth
  recording); < 0.85 = debt real → release a moving-frame head-retrain knob.
  Either way a finding; escalation stops at heads.
- **C4-P0 — roomgraph_v2 offline replay (1–2 d).** On the existing ~935
  passage-firing dataset (69 % box-pinch false), score a *traversal-window*
  geometric feature (flanking structure persists on BOTH sides through a
  crossing; a passed box flanks one side). Windowed AUC ≥ 0.85 (vs single-frame
  0.631) releases the flight regate; 0.75–0.85 releases a learned window-head
  knob (still geometry — no WM); < 0.75 → record the honest bound, room graph
  stays clean-room-only.
- **Indoor-gate scaffolding + ceiling probe (1 d + overnight).** Build the
  seeded mission generator (below), run n=20 per family as the
  feasibility-first ceiling probe — bars are frozen from this probe, never
  from hope (the slalom lesson).
- **Housekeeping (1 h).** Propose the v0.8.0 tag + a changelog entry for the
  person arc (owner decides both).

### Horizon 2 — 1–3 months: one perception campaign in the foreground, one training campaign in the background

- **INDOOR-GATE v1 (the missing 72/100; ~1.5 wk elapsed).** A seeded mission
  generator draws from a weighted pool — M1 single-room find+return, clutter
  tier 0/1 (0.30); M2 multi-room 2–4 rooms (0.30); M3 vertical target ∈ {0.3,
  1.0, 2.0 m} with multi-height yaw scan (0.20); M4 person find+return (0.20).
  Clean + clear-lane clutter only; **speed 0.6 and beams8 hardcoded in the
  runner** (do not route through `eval_search.suite()` until I1 lands).
  Composite verdict like transit: found AND returned AND zero collisions, with
  a per-family break histogram. Prior from unit numbers (frozen only after the
  ceiling probe): composite ≥ 0.80, pooled collision ≤ 0.02, return|found
  ≥ 0.90, n=100 seed-drawn missions, passing-seed videos of record. Note: M3's
  "search the rooms, then scan heights" end-to-end has **never been measured** —
  the gate's first new number. Then `scripts/gate.py`: one command → transit
  gate + indoor gate + budget selftest + safety suite + lock check → one JSON
  scorecard per mode. Quarterly cadence + after any WM/head/planner touch.
- **C2 — evidence_v1 (3–5 d, offline-first).** Detection as sequential
  evidence: calibrated per-frame log-likelihood ratios accumulated SPRT-style,
  with a calibrated "not present, move on" abstention — against the incumbent
  ad-hoc rules (debounce traded FA 0.95→0.35 for miss 0.40 in `vision_v1`;
  confirm-2 in `person_v1`). Phase-0 is a pure replay duel on C1's recorded
  streams (no flights): SPRT vs confirm-k on identical data. Only the winner
  gets choreographed and regated on the indoor gate. Bars: person find ≥ 0.93
  held AND false-claim ≤ 0.05 AND median decisions-to-find −20 %. Kill: if
  confirm-k ties SPRT offline, record that the heuristic was already optimal
  and export the calibration tables. → **article #9**.
- **C7 — person_real_v1 (2–3 d; the C8 trigger).** Posed multi-primitive
  person proxies: standing / sitting / crouching / lying / half-buried under a
  slab. Frozen-latent + retrained-head AUC per pose, table frozen up front:
  standing ≥ 0.90, sitting/crouching ≥ 0.80, **lying = the go/no-go at 0.75**.
  Every cell is a finding. Prediction on record: lying-from-a-level-camera
  fails — and that pre-registered failure is what arms the pitch step (C8).
  If lying passes, C8's retrain loses its justification — the cheap outcome.
- **K-series — "both high" swap robustness (background; 2–3 wk elapsed).**
  The named next step of `slalom_stopobserve_v1`. One knob each: K1 aug ratio
  0.50→0.66 champion-weighted at 500 k (bars: champion ≥ 0.70 AND unified
  ≥ 0.50, n=20); K2 budget 500 k→1.0 M at the K1 winner (bars: ≥ 0.75 / ≥ 0.70);
  K3 conditional (arbitrate with `research step`, never `run`): stop_hover
  stacked on the best aug config. Kill: K2 fails → record "swap-invariance for
  long-chain skills costs a re-distill per encoder" — a priced negative that
  validates Option A permanently. → **article #12** either way.
- **P1 — indoor sensor pricing (1–2 d).** The entire indoor safety case rides
  beams8 and **no beam has ever been noisy**. Inject range noise, ToF-style
  dropout-to-max, and 0–2 decision-steps latency at the `beam_ranges` seam;
  `shift_appearance` on detection frames. Read: detection AUC + n=30 mission
  subset vs perturbation level. The corridor veto's margins are the number
  under test.
- **P2 — scan-compounding stress (1–2 d).** Motion blur during yaw, 2×
  decision interval, threshold sweep — 4–6 pre-registered cells, n=15–30.
  Prices the per-frame→mission conversion honestly.
- **Safety selftest suite (2–3 d).** One deterministic scenario per
  sim-gateable row of `docs/safety.md`: geofence assert, vertical envelope
  (turn lowfly's measured behavior into asserts), imminent backstop (wall
  dead-ahead, veto must fire), emergency-land choreography (capped descent to
  z≈0.05 from any mission state — the one small new behavior), low-battery
  return (decision-budget proxy; label honestly: tests choreography, not
  battery physics), log replay (trajectory match within tolerance — doubles as
  the hardware black-box seam). Manual override / GO-NO-GO stay honestly in
  the hardware column. Fast rows → CI; closed-loop rows → the scorecard.
- **Infra pre-requisite for H3:** wire `kind="world_model"` into the research
  runner (`scripts/research.py` currently rejects it; the ledger's own trigger
  — "a second hand-queued model-axis campaign" — fires with C5).
- **Publish #8** (bilingual) once Q1's parity table exists.
- **The quarter's gate: C5 Phase-0.** The oracle-memory probe (~2 d, zero
  training): append a privileged "recently-seen, now-out-of-FOV occupancy
  summary" to the policy/safety inputs in dense transit. If privileged memory
  does not cut dense crash by ≥ 5 pts, memory is not the binding constraint
  and the retrain dies for free. If it passes: full pre-registration (with the
  mechanism argument from Part II) → **owner GO/NO-GO at end of month 2**.
