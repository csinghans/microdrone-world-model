## 第二部 — 論題：從瞥見到證據

空間的分工已經定案且量測完畢。看看還開著的問題，每一條前沿都關於
**時間**、而不是空間：

- per-frame recall ~0.5–0.76 是靠*編排*（掃描旋轉、confirm-2、接近後
  面對確認）複利成任務 recall ~0.93 的——臨時湊的時間整合，從未形式化
  （`vision_v1`、`yaw_v1`、`person_v1`）；
- 一幀分不出門口和箱縫（AUC 0.631）——被點名的候選是*軌跡整合*的結構
  （`search_roomgraph_v1`）；
- dense transit 前沿的殘餘死因是 FOV 邊緣的側雜物，×2.0 下 13/14——
  **滑出視野的東西**（`experiments/dense_speedrun/`）；
- 停下來觀察對連續 policy 是 OOD（`slalom_stopobserve_v1`）；
- dispatch 弧的結語：world identity 需要*探測運動*——只存在於跨幀之間
  的資訊（`experiments/dispatch/`）。

所以本計畫唯一的科學主題：**把瞥見變成證據。** 一階一階來：先形式化
決策層的證據累積（零重訓、先把判官蓋好），然後——且只有通過 oracle
探針與 owner 之門——才是本視野內唯一一次 WM 重訓：把記憶放進模型本體。
主題會複利：同一個動作（單幀 → 整合軌跡）同時適用於相機通道（C2）與
beam 通道（C4）；temporal WM（C5）若活過它的探針，正好餵給 conditional
calibration 它缺的 context 訊號；而重蒸餾機器（K 系列）是讓任何 WM 改進
都能出貨、不再把 zoo 晾在原地的保險。

**關於重開 GRU。** v0.2 的負結果仍然成立（就在禁區表裡）。temporal
戰役只能以*帶機制論證的重開*進場，照家規（`RESEARCH-IDEAS.md`
chain-algorithm 條目）。論證已預先寫好：(a) 當年的判官是單 seed 模型軸
AUC——後來量測到每 seed 散佈 ~0.5 的壞儀器；(b) 現在手上的失效簽名
（FOV 邊緣死因、瞥見受限的 recall）恰好是「出了畫面的資訊」，任何
非遞歸的 stack 原則上無法表徵；(c) 這次給的資料規模是 v0.2 GRU 的
5–10 倍。下面的 C5 被設計成：如果連特權記憶都幫不上忙，這次重訓就
免費死掉。

## 第三部 — 計畫

三個視野，純模擬。每條戰役都走家規的形狀：Phase-0 探針 → 凍結 bars →
判詞（怎樣都是 finding）。成本以 Mac-天計；「背景」指帶 `*-DONE` 標記的
隔夜佇列。

### 視野一 — 0–2 週：先蓋儀器、再談意見

全部零 WM 風險、全部平行安全，每一項都產出一個數字或一件儀器。

- **I1 — eval 預設稽核（0.5 天）。** 把 `INDOOR_SPEED = 0.6`＋可部署
  safety 名稱提升為單一常數模組；讓 `eval_search.suite()` 預設用它；
  selftest 斷言 CLI 與 API 預設一致；每個 suite 印出 config 標頭。然後
  grep 稽核所有 eval 的預設是否符合其文件化的 robust 配置。
- **G1 — 頭治理（1 天）。** `artifacts.lock.json` v1.1（增量式）：把三顆
  *部署中*的頭（`yaw`、`low`、`person`）鎖進去，新增選用欄位 `wm`
  （頭訓練所在 latent 的 sha）與 `track`；`fetch_champions` 的超集檢查
  本來就容忍多餘鍵。`FlightMode` 增加頭綁定＋`verify()` 對鎖交叉檢查；
  CI 安全的一致性 selftest（無 artifacts 也能跑）。更新 release，讓
  fresh clone 能還原**完整的飛行系統**。附帶一個 owner 決定：`alt` vs
  `alt_os` vs `low` 正名（eval 路徑用的是 `low`）。
- **Q1 — int8 parity 探針（2–3 天；writing/#8 缺的核心量測）。** 手寫
  ~150 行 fake-quant 包裝（透明度勝過 `torch.ao`）：權重 int8——conv
  per-channel／linear per-tensor，激活 int8 per-tensor，bias int32——
  對齊 GAP8 NNTool SQ8 流程；另報一個 per-tensor 權重 arm 作悲觀下界。
  校準用**兩軌**各 512–2048 幀（統一 WM 只用一軌校準會說謊）。量化後
  重跑一模一樣的 gates：held-out AUC@32、一格閉環 transit（n=60 同
  seeds）、室內偵測 AUC、n=30 室內任務子集。**Bars 在任何數字出現前
  凍結**：ΔAUC@32 ≥ −0.010；Δcrash ≤ +3 pts 且落在浮點 run 的二項 CI
  內；頭 ΔAUC ≥ −0.015。預註冊的診斷：逐層量化 SNR 點名敵對層；決策
  規則——投影／池化層敵對 → 加寬校準；predictor 殘差累積敵對 → 僅
  predictor 用 int16 激活（標記為橋接問題）；頭敵對 → 在量化 latent 上
  重訓頭（幾秒的事）。加一個**灰階輸入 arm**（AI-deck 原廠感光元件是
  單色；stack 用 RGB 訓練）——用零頭錢為那個缺口定價，完全落在
  「探針級 sim-to-real」的許可內。誠實清單（模擬無法驗證的）：PULP-NN
  捨入的精確性、Autotiler 的切磚／DMA 與真實延遲（0.5 GMAC/s 本來就是
  「寫出來等人挑戰」的數字）、NNTool 實際的逐層方案、真實相機。
- **C1 — detect_inflight_v1（1–2 天）。** 償還「靜態成像」的債：讓搜索
  runner 在 speed 0.6 的飛行*當中*記錄逐幀頭輸出＋真值可見性；replay
  打分。預註冊讀法：飛行中 pooled AUC ≥ 0.90 = 債已還（值得記錄的誠實
  正結果）；< 0.85 = 債是真的 → 釋放 moving-frame 頭重訓旋鈕。怎樣都是
  finding；升級止步於頭。
- **C4-P0 — roomgraph_v2 離線 replay（1–2 天）。** 在既有 ~935 筆
  passage-firing 資料（69 % 是箱縫誤觸發）上，先給*穿越視窗*幾何特徵
  打分（門口的側翼結構在穿越全程兩側都在；路過的箱子只在單側）。視窗
  AUC ≥ 0.85（vs 單幀 0.631）才釋放飛行 regate；0.75–0.85 釋放 learned
  window-head 旋鈕（仍是幾何——不用 WM）；< 0.75 → 記錄誠實上界，
  房間圖維持淨空房限定。
- **室內門鷹架＋天花板探針（1 天＋隔夜）。** 蓋好種子化任務產生器
  （見下），每 family 跑 n=20 作 feasibility-first 天花板探針——bars 由
  探針凍結，不由願望凍結（slalom 的教訓）。
- **Housekeeping（1 小時）。** 提案 v0.8.0 tag＋person 弧的 changelog
  條目（兩者皆 owner 決定）。

### 視野二 — 1–3 個月：前景一條感知戰役、背景一條訓練戰役

- **INDOOR-GATE v1（缺席的那個 72/100；~1.5 週經過時間）。** 種子化
  任務產生器按權重抽樣——M1 單房 find+return、雜物 tier 0/1（0.30）；
  M2 多房 2–4 房（0.30）；M3 垂直目標 ∈ {0.3, 1.0, 2.0 m}＋多高度 yaw
  掃描（0.20）；M4 person find+return（0.20）。只收 clean＋clear-lane
  clutter；**runner 硬編碼 speed 0.6 與 beams8**（I1 落地前不得繞經
  `eval_search.suite()`）。合成判準與 transit 同款：found 且 returned
  且零碰撞，附 per-family 破口直方圖。單元數字給的先驗（僅在天花板
  探針後凍結）：composite ≥ 0.80、pooled collision ≤ 0.02、
  return|found ≥ 0.90，n=100 種子抽樣任務、過關 seed 錄影存證。注意：
  M3 的「先搜房、再掃高度」端到端**從未被量過**——這是門的第一個新
  數字。然後 `scripts/gate.py`：一個指令 → transit 門＋室內門＋預算
  selftest＋安全套件＋lock 檢查 → 每模式一份 JSON scorecard。季度
  節奏＋任何 WM／頭／planner 變動之後必跑。
- **C2 — evidence_v1（3–5 天，離線先行）。** 偵測=序貫證據：頭輸出
  校準成逐幀 log-likelihood ratio、SPRT 式累積，加一個校準過的「不在、
  換地方」棄權——對打現任的土法（`vision_v1` 的 debounce 用 FA
  0.95→0.35 換來 miss 0.40；`person_v1` 的 confirm-2）。Phase-0 是純
  replay 對決（不飛）：SPRT vs confirm-k 在 C1 錄的同一批流上。贏家才
  進編排、上室內門 regate。Bars：person find ≥ 0.93 保持、false-claim
  ≤ 0.05、decisions-to-find 中位數 −20 %。殺標：離線就打平 → 記錄
  「土法已然最優」並輸出校準表。→ **文章 #9**。
- **C7 — person_real_v1（2–3 天；C8 的扣板機）。** 多基元擺姿的人形
  proxy：站／坐／蹲／躺／板下半掩。凍結 latent＋重訓頭的逐姿態 AUC，
  表格事前凍結：站 ≥ 0.90、坐蹲 ≥ 0.80、**躺 = 0.75 的 go/no-go**。
  每一格都是 finding。預測先押在紀錄上：水平相機看躺姿會崩——而那個
  預註冊的崩，就是武裝 pitch 步（C8）的量測。若躺姿過了，C8 的重訓
  就失去理由——便宜的好結局。
- **K 系列 — 「兩邊都高」的 swap 穩健性（背景；2–3 週經過時間）。**
  `slalom_stopobserve_v1` 點名的下一步。一次一顆旋鈕：K1 aug 比例
  0.50→0.66 冠軍加權、500 k（bars：冠軍 ≥ 0.70 且統一 ≥ 0.50，n=20）；
  K2 預算 500 k→1.0 M 用 K1 贏家（bars：≥ 0.75／≥ 0.70）；K3 條件釋放
  （用 `research step` 仲裁、絕不用 `run`）：最佳 aug 配置疊 stop_hover。
  殺標：K2 敗 → 記錄「長鏈技能的 swap 不變性，代價是每換一次 encoder
  重蒸餾一次」——一個把 Option A 永久正當化的標價負結果。→ 不論勝敗
  都是**文章 #12**。
- **P1 — 室內感測器定價（1–2 天）。** 整個室內安全案騎在 beams8 上，
  而**沒有任何一束 beam 加過噪聲**。在 `beam_ranges` 縫注入距離噪聲、
  ToF 式 dropout-to-max、0–2 決策步延遲；偵測幀套用既有
  `shift_appearance`。讀數：偵測 AUC＋n=30 任務子集 vs 擾動等級。
  受試的正是 corridor veto 的裕度。
- **P2 — 掃描複利壓測（1–2 天）。** yaw 中的動態模糊、2× 決策間隔、
  閾值掃描——4–6 個預註冊格、各 n=15–30。誠實地為 per-frame→mission
  的轉換定價。
- **安全 selftest 套件（2–3 天）。** `docs/safety.md` 中每一條模擬可
  gate 的列各配一個確定性場景：geofence 斷言、垂直包絡（把 lowfly 的
  量測行為變成斷言）、imminent 後盾（正前方牆、veto 必須觸發）、緊急
  降落編排（任意任務狀態限速降到 z≈0.05——唯一一個小的新行為）、
  低電回航（電量=決策預算 proxy；誠實標註：測的是編排邏輯、不是電池
  物理）、log replay（軌跡容差內重現——同時是硬體黑盒子的縫）。手動
  接管／GO-NO-GO 誠實留在硬體欄。快的列進 CI；閉環的列進 scorecard。
- **H3 的基建前置：** 把 `kind="world_model"` 接進 research runner
  （`scripts/research.py` 目前拒收；帳本自己的觸發條件——「第二個
  手排的模型軸戰役」——在 C5 上點火）。
- **發表 #8**（雙語），在 Q1 的 parity 表存在之後。
- **本季之門：C5 Phase-0。** oracle-memory 探針（~2 天、零訓練）：把
  特權的「剛看過、現已出 FOV 的佔用摘要」接到 dense transit 的 policy／

### 視野三 — 3–12 個月：唯一一次重訓、晉升決策、北極星

- **C5 — temporal_wm_v1（owner-gated；背景 ~2 週牆鐘）。** 僅在

  ≥3 seeds：dense 墜機 ≤ 12 %（pooled n≥200）；guards 凍結（classic／

  → 維持雙 WM 並成文：`flight_mode` 斷言駐留 WM int8 總和 ≤ 200 KB 且

  驗證過的零重訓路）；**新技能一律只在統一 latent 上訓練**，讓冠軍的
