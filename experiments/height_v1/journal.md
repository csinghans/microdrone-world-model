# height_v1 — 室內空間高度:幾何測距(不是 WM)

## 背景與判斷(Hans:「加 pitch 量室內高度嗎?」)

問題是能不能加 pitch 讓 WM 量室內高度。查證後的誠實判斷:**pitch 不像 yaw**——
(1)四軸的 pitch 是它平移的致動器(PID 靠俯仰產生水平運動),不能像 yaw 那樣原地
定俯角往上看;(2)WM 從沒看過俯仰/垂直畫面,對它是 OOD(yaw 偵測不變是特例——目標
在前方的畫面朝哪都一樣;天花板在上方是全新畫面);(3)**量高度是度量/幾何任務**,
正是整條室內弧的定律裡「WM 輸給便宜幾何」的那一類(避撞→雷測、覆蓋→格)。所以高度
該用**向上測距**,不是單目 WM。Hans 選「先做幾何版」。

(另外:sim 原本是開頂 2D 平面圖,根本沒有天花板可量——得先加垂直結構。)

## 做法(加法、不破壞既有)

- `SearchScenario` 加 `ceiling_h`(預設 0.0 = 開頂,**每個既有 2D 房間不變**)、
  `beam`(低樑 = 一塊較低的天花板 patch)。`single_room(ceiling=True)` 才給每個 seed
  一個天花板高(2.0–3.2 m)+ 一段低樑;預設 `ceiling=False`,搜索/偵測/覆蓋全不受影響。
- `spawn_bodies`:`ceiling_h>0` 時多生一片天花板 slab + 低樑;**天花板帶碰撞形狀**
  (不像視覺-only 的牆),好讓向上射線打得到。
- `ceiling_range(env)`:**向上 rangefinder**——從無人機上方一點朝上 `rayTest`,回傳
  到天花板的淨空(可佈署的 $5 向上測距的模擬)。`ceiling_at(x,y)`:解析真值。
- `eval/eval_height.py`:座標偏移把房間移到位、無人機停 START、向上射線量測 vs 真值,
  跨網格位置(含低樑)。

## 結果 — 幾何測高完美

8 房、326 取樣點:

| 指標 | 值 |
|---|---|
| MAE | **0.0 cm** |
| max error | 0.0 cm |
| 樑下淨空 | 0.62 m |
| 開闊淨空 | 1.63 m |

向上測距**精確**恢復天花板高(sim 幾何本就精確),且**映出非平坦剖面**——樑下淨空
明顯低於開闊處(0.62 vs 1.63 m),等於偵測到「低淨空危害」。

## 判詞 — 高度是解掉的幾何測量,不是 WM 的活

**可佈署的室內測高 = 一根向上 rangefinder,cm 級、映出天花板剖面(含低樑),便宜、
準、直接。** 這是對的工具,跟 beams8 之於避撞、格之於覆蓋同一個道理:WM 贏感知、輸
度量/空間。拿單目 WM 去估度量高度是在跟它的本性作對(且要 pitch-view 重訓)。

## 誠實邊界 / 下一步(未做,Hans 拍板)
- sim 的 rayTest 對精確幾何當然是 0 誤差;真硬體的向上 ToF/測距有雜訊,但那是感測器
  規格、不是方法問題——方法(向上測距)是對的。
- **WM 在「有高度」的世界裡的角色仍是感知**:認出垂直結構(低樑、夾層、上方有無物體)
  ——那才需要 **pitch-view 的 WM 重訓**(真重訓,pitch 是 OOD、不像 yaw 免費)。這是
  比 yaw 大的一步,留給你要走時再拍板。

  要更真實可注入測距雜訊/量化。
