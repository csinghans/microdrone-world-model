# 打造動作條件化的 latent 世界模型（權重只有 81 KB）

*本文是「把世界模型塞進 512 KB」系列第四篇。前三篇講為什麼；這一篇講
怎麼做——每個設計決定、背後的實測理由，以及輸掉的那些選擇留下的傷疤。
全部可在
[microdrone-world-model](https://github.com/csinghans/microdrone-world-model)
重跑。*

---

> **白話版。**老司機不會記得路上的每一幀畫面——他帶著一份精簡的「路感」，加上每個動作會通往哪裡的直覺。我們把這件事做進 81 KB 的權重裡：把相機壓成 64 個數字、學會每個指令會怎麼改變這些數字、再問一個小小的裁判「這樣飛下去會不會撞」。下面每一個設計決定都有同一個理由：另一個選項實測飛得更差。


## 零件清單

整疊系統是四個小網路加一位特權教師：

```
鏡頭 64x64 ──> Encoder（64 維 latent z，帶方位感）
                  │
   指令 a ───────┤   Predictor: ẑ(k) = z + Δk(z, a)，k = 4/8/16/32 步
                  │
                  ├─> 碰撞頭：P(0.7 m 內)、P(0.35 m 內)
                  │       每個視野一組——規劃器讀的就是這個
                  └─> Danger-Now 頭：「現在就很近？」——誠實的
                          反應式基線，同一雙眼睛

  （僅訓練期）Encoder 的 EMA 雙胞胎——永不上機的評分老師
  （僅訓練期）反事實 oracle——懂幾何的判官
```

權重：**81.3 KB int8**。加上 activations 與工作區：**137.3 KB**、
約 390 萬 MACs ≈ 每次決策 8 ms（GAP8 級晶片）。以下所有內容都在讓
這些 KB 回答同一個問題：*我的哪個候選指令會靠近什麼東西、什麼時候？*

## 資料是實驗，不是影片

「一幀對一標籤」的映射可以從被動影片學；世界模型不行——它得學世界
*因為你的動作*而怎麼變。所以資料集是自己飛出來的一串小型對照實驗：
重置到全新的隨機柱林、巡航進場，然後每約一秒從六張指令卡（前進／左閃
／右閃／減速／爬升／懸停）抽一張並**堅持執行整段**。96 趟 × 120 步
@ 48 Hz，每趟各自的巡航速度。

「堅持」的價值有兩層。第一：模型要回答「如果我*維持*這個動作 k 步會
怎樣」，訓練配對就必須真的有維持了 k 步的動作。第二：每次換卡都是
天然對照——同樣的視野、不同的指令、不同的下場——這正是規劃器要租用的
動作條件化。

## 脊椎：三個選擇、三個理由

**Encoder 帶方位感。** 無人機的決策變數不是「有沒有柱子」而是「柱子
在我視野的哪裡」——方位才分得開左閃和右閃。所以 encoder 的池化保留
水平結構而不是平均掉它。全域平均池會把「柱在左」「柱在右」壓成同一份
摘要；我們的做法讓這個差別一路活到那 64 個數字裡。

**Predictor 是殘差式。** ẑ(k) = z + Δk(z, a)：「未來長得像現在」是
免費起點，網路只掙*差值*。81 KB 養不起重學恆等函數的模型——而且在
48 Hz 下，世界大部分時刻真的就是恆等。（這個選擇後來還救了我們一次
真 bug；傷疤故事見下。）

**碰撞頭用雙環讀預測的未來。** 每個視野兩個 logit：P(0.7 m 內) 和
P(0.35 m 內)。單環是區域測試、不是風險梯度——進了警戒環之後，*每個*
指令的警戒標籤都是 1，連逃出去的那個也是。危險環讓規劃器在警戒環
飽和的地方仍然看得見。這不是理論上的講究：後來教策略穿窄縫時，縫內
警戒環整片飽和、只剩危險環帶著可用訊號——雙環設計在那裡通過了它的
實戰大考（穿縫成功率 27 % → 87-90 %）。

## 永不上機的評分老師

誰給 predictor 打分？不是像素——是 encoder 的**EMA 雙胞胎**（權重的
指數移動平均、stop-gradient）把*真實*未來幀也編成 latent，predictor
以「落點離這個目標多近」計分。

為什麼要雙胞胎、不用本尊當靶？因為活靶會追著學生一起塌縮：encoder
只要把所有畫面都編到同一個點，每個預測就都「正確」了。兩道防線一起
上：EMA 目標動得夠慢、足以當穩定的答案卷；**變異數守衛**（VICReg 式
hinge）強制 latent 各維度不得躺平。便宜、非生成式、防塌縮——JEPA 配方
的奈米版。

## 教會「排序」的判官

這是最重要、也最容易做錯的監督。實飛資料回答「我飛的那條路有沒有
靠近？」——密集、真實、而且對規劃器幾乎沒用，因為規劃器問的是「從
這裡出發，我的六個選項哪個最安全？」實飛只在換卡瞬間回答這題。我們
量過這個缺口：只用實飛標籤，碰撞 AUC 看起來不錯（~0.9），但模型
「把左閃和右閃排出高下」的能力停在隨機水準。好偵測器、擲硬幣的選擇者。

解法是模擬器唯一誠實的超能力：**反事實 oracle**。對每一幀、每一個
候選指令，用直線運動學把指令穿過已知柱林往前推，標記各視野內會不會
進入各環。雙方都是直線近似——v0.2 起有運動感知，橫越的柱子以*相對*
速度判——且近似對所有候選對稱，所以排序在近似下存活。

兩條誠實條款隨行：

1. **FOV 遮罩。** 威脅柱在鏡頭楔形外的題目標記「不可作答」、不進
   loss。拿單眼模型看不到的威脅考它，教出來的是噪音不是警覺。
2. **決策相關過採樣。** 大多數幀離什麼都遠、每個候選都安全、無從
   排序。反事實 batch 的一半從「可見候選之間有分歧」的幀抽——真正
   教「選擇」的那些幀。

## 訓練配方，以及動它的代價

五項 loss 相加：latent 預測 MSE、變異數守衛、實飛碰撞 BCE、反事實
BCE（FOV 遮罩）、danger-now 頭。Adam 80 輪，筆電上約半小時。

這個配方是一個*耦合系統*——要幫自己蓋世界模型的人，我最想畫線的就是
這段。兩道我們自己 changelog 裡的傷疤：

**傷疤一：吃掉基線的記憶。** 給模型加 GRU 記憶（過去 latent 的摘要）
時，最順手的接線是把 GRU 狀態交給 predictor 當殘差基底。這一下悄悄
把「未來長得像現在這*幀*」換成了「未來長得像我的*記憶*」——免費基線
不再免費。煙霧測試當場抓包：長視野預測誤差*比什麼都不做還糟*。修法
成了一條設計規則：**記憶可以當預測的條件；殘差必須留在幀上。**
（修好之後，GRU 還是誠實地輸掉了它的閘門：它恰好在記憶不是瓶頸的
場景幫忙、在兩個難場景扯後腿。堆疊歷史的策略保住王座。在這裡，優雅
必須付房租。）

**傷疤二：讓飛行變爛的輔助 loss。** 今年我們加了一個幾何上無懈可擊
的輔助監督：用度量佔用網格教 latent「東西在哪」（完整故事在第二篇）。
偵測在每個切片上都變好——而在那顆模型上重訓的策略，在密集雜物裡撞毀率
*翻倍*。碰撞頭吐出的機率曲面就是策略的感官輸入；多一道梯度打進共用
主幹，就把曲面重新校了一遍（後來量到是條件性的——任何全域旋鈕都修
不了）。教訓可以推廣：**你加的每一項 loss，都會重塑模型下游消費者的
觀測空間。** 加 loss 要像加藥一樣加。

## 不騙自己的驗收

三個習慣，全都便宜、全都承重：

- **按趟切分，永不按幀。** 相鄰幀近乎重複；隨機按幀切等於把考卷洩進
  訓練集，每個指標都會虛高。
- **考規劃器真正租用的那一項能力。** 閃避排序考：在保留幀上，幾何
  明確一側較安全（一側穿越危險半徑、另一側淨空 >0.12 m、威脅在 FOV
  內）時，模型是否排對？隨機是 0.5。我們**每次重訓 draw 都是 1.00**
  ——這是整個 repo 裡我最信任的一個數字。
- **切片驗收，然後不信小切片。** 分世界 AUC 告訴我們密集雜物是開放
  前沿；它也教了更狠的一課：在 18 趟的驗證切分上，*單一 seed* 的
  dense AUC 在同配方重訓之間可以從 0.47 擺到 0.99。模型層閘門現在
  一律要求多 seed 均值，或干脆直接以飛行閘門為準。

## 成本表

| 項目 | 價格 |
|---|---|
| encoder＋predictor＋雙環頭（權重）| 81.3 KB int8 |
| 含 activations＋工作區的完整預算 | 137.3 KB／512 KB |
| 決策延遲 | ~390 萬 MACs ≈ 8 ms |
| 訓練 | 96 趟自飛資料＋80 輪 ≈ 筆電量級 |
| oracle 與 EMA 雙胞胎 | 部署免費——它們從不出實驗室 |

這個設計塞得下的原因：部署期的一切都小，昂貴的一切都是*老師*而非
乘客。重監督、輕推理——我認為這是整個 build 裡最值得帶走的一個想法。
