# 用撞毀率評估，不用 loss

*本文是「把世界模型塞進 512 KB」系列第五篇，也是整個系列的論文主旨：
其他每一篇文章的數字所站立的評測紀律——以及它四次實測救我們免於
出貨更爛東西的記錄。全部可在
[microdrone-world-model](https://github.com/csinghans/microdrone-world-model)
重跑。*

---

> **白話版。**模擬考分數更高的學生，路考照樣可能被當。我們升級過在每一項紙面指標上都更漂亮的模型——飛起來反而更糟。這個專案唯一信任的分數，是唯一算數的那個：真任務上有沒有少摔？這條規則有四次量測在案，四次都攔下了「包裝成升級的降級」。


## 那筆我們差點買下的升級

今年我們給無人機的世界模型加了度量監督——「東西在哪、離多遠」。從
模型層的每一個指標看，這筆升級都是乾淨俐落的大勝：密集場景的碰撞
AUC 對同資料對照組**+0.07 到 +0.24**、反應式頭 +0.13、閃避排序探針
*翻倍*。如果我們的驗收是 loss 曲線和 AUC——也就是大多數模型升級得到
的那種驗收——當天下午就出貨了。

然後我們在升級後的模型上重訓飛行策略，配方完全不動、單一變因，飛
標準記分板：密集雜物的撞毀率**巡航速 17 % → 37 %**、高速
**27 % → 48 %**（n = 60、全新種子）。模型懂得更多，無人機撞得
加倍勤。這筆升級沒有出貨的唯一原因是：我們的驗收以**撞毀**計價，
不以模型統計量計價。

這是這個 repo 第四次量到同一句話的某個版本——**更好的偵測器不是
更好的飛行**——而且是最鋒利的一次，因為這回偵測器和判決之間隔著一整
次從零重訓的策略，增益照樣兌現不了。

## 代理指標為什麼脫鉤：三個實測機制

Loss 和 AUC 是關於*模型自己、在資料集上*的統計量。飛行是一個
*閉環*的性質：模型、策略、環境、回饋。我們實測過它們分道揚鑣的三種
方式：

**1. 排序不等於校準。** AUC 只看*順序*——對機率數字本身是多少
渾然不覺。但學習出來的策略吃的是碰撞頭的原始機率曲面。我們的度量
接地升級讓順序更對、同時把數字*灌水*——而且是條件性的、專挑密集
幾何灌（dense 警戒環的校準誤差惡化 1.75 倍），任何全域修正都無從
表達。策略的觀測貨幣被悄悄貶值，成績單卻更好看了。

**2. 固定決策門檻假設地景永凍。** 我們第一代規劃器是手調機率門檻的
latent-MPC。它先後被打壞三次——從來不是被更爛的模型，每次都是被
*更好*的模型：重校過的機率從門檻底下移走了（第三振：一次運動感知
重訓首度讓移動障礙變得可偵測，同時把手調規劃器的密集場景撞毀率推到
90 %）。手調門檻是在賭機率地景永不變動；它每次重訓都在動。

**3. 平均藏切片；切片決定任務。** 一個模型側記憶（GRU）曾讓總表
驗證 AUC 上升——靠著幫簡單世界（classic 0.86 → 0.95）、同時傷害兩個
難世界（dense 0.82 → 0.74、moving 0.88 → 0.84）。平均值說進步；
任務格子說退步。它誠實地輸掉了自己的閘門，更簡單的架構保住王座。

## 我們改飛什麼：記分板

能同時抓住這三種失敗模式的驗收，無聊而具體——一張**任務格子**表，
每格是一組（世界 × 速度）的閉環飛行：

| 慣例 | 理由 |
|---|---|
| 每格撞毀率，n = 30 種子 | 任務自己的貨幣 |
| 所有策略共用同樣種子 | 差異來自策略，不是運氣 |
| 你宣稱改善的格子設 **targets** | 改動的意義所在 |
| 其餘所有格子設 **guards** | 贏了卻破 guard＝洞搬家了 |
| 標準在任何數字存在*之前*凍結 | 不許事後搬門柱 |
| 壓線（±0.08）→ 全新種子 n = 60 重測 | 見下——這條規則值回票價 |
| 負結果照實記錄，永不重試到通過 | 日誌本身就是產品 |

這套跑起來很便宜（一格＝30 次模擬飛行）、吵起來很無情。我們的自主
研究 runner 執行的正是這套協議——預註冊、閘門、重測、日誌——戰役的
紀律從此不再取決於凌晨一點某個人類的意志力。

## 儀器課：連撞毀率都需要誤差棒

飛行指標是對的貨幣，不是魔法貨幣。兩則實測的謙卑：

- 一個密集格子第一批三十顆種子讀 **26.7 %**、六十顆全新種子讀
  **48.3 %**。友善的首讀落在預註冊的壓線帶內，重測規則自動觸發，
  判決從「有得吵」翻成「無可爭辯」。有一場戰役總共觸發了**七次**
  這樣的重測。如果你的驗收沒有重測規則，你的標準比看起來軟。
- 模型層的切片指標更糟：*同配方、同資料*的五次重訓之間，單一 seed
  的 dense AUC 從 **0.47 擺到 0.99**。我們現在把任何單 seed 模型
  指標當天氣、不當氣候——多 seed 均值，或直接以飛行閘門為準。

## Loss 還有什麼用

我們沒把訓練曲線丟掉；我們把它降職了。Loss、AUC 和煙霧測試回答的是
*工具*問題：頭死了沒、latent 塌了沒、佈線通不通、預算裝不裝得下。
我們的煙霧斷言被限縮到微型訓練真能承諾的範圍——所有關於*品質*的
主張一律轉呈飛行閘門。（這個限縮也是付過學費的：某個已出貨版本的
煙霧測試後來被發現斷言了煙霧量級給不起的東西，而且在一個從來沒人
跑過的 CI 工作裡默默紅著。重校後的版本現在每週在公開 CI 上綠燈。）

## 帶得走的配方

如果整篇只帶走一節，帶這節：

1. 寫下任務格子——你的系統必須活下來的（情境 × 條件）矩陣。
2. 在任何數字存在之前凍結 targets **和 guards**。guards 神聖不可
   侵犯：它們是你發現洞在搬家的方式。
3. 每個候選者飛同樣的種子。發表跨重訓的範圍，不發表你最好的一次。
4. 給壓線結果一條自動的、預先註冊的重測規則（更高 n、全新種子）。
5. 負結果和勝利記在同一本日誌。今天花你一次訓練的負結果，就是明年
   沒人重挖的那個坑。
6. 模型指標拿來除錯；任務指標拿來裁決。

Loss 是模型的感受；撞毀是任務的感受。用任務的貨幣驗收，那些報表
更漂亮、飛起來更爛的模型就會被擋在閘門口——而過去四次，我們的正是
在那裡被擋下來的。
