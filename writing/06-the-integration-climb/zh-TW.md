# 單元測試全綠，整合測試全紅：爬向佈署門檻的五套陣容

*本系列談的是把世界模型塞進 512 KB 的微型無人機。第 1–5 篇在打造與
審計單一技能；這一篇談把技能串起來之後的事——一場全目錄第一天就
考砸的飛行大考，以及最後通過它所花的五套陣容。所有數字都能從
[microdrone-world-model](https://github.com/csinghans/microdrone-world-model)
重跑出來。*

---

## 一場沒人考過的考試

到 2026 年年中，這個 repo 有五個飛行技能，各自帶著把關過的戰績：
穿靜態縫、繞三柱蛇行、抓移動縫的時機、過門、等一扇正在打開的門。
用軟體工程的話說，我們有**單元測試**——而且是異常嚴格的那種：每個
技能在凍結種子上對著預先註冊的及格線應考、新技能訓練時舊技能有
guard 保護、任何數字都不准事後補記。

我們沒有的，是每個軟體團隊遲早要說出口的那句話：*單元全過，產品
是壞的*。從來沒有任何工件連續飛過兩種不同的障礙。

所以我們把缺的那一層補上，並在量任何數字之前凍結規則。**整合測試**
＝從五技能池隨機抽三關（可重複）組成一條賽道，端到端飛完——約四十
個控制決策、關與關之間不重置。**佈署門檻**——上實機的常設前置條件
——是 **n = 100 條隨機賽道成功率 ≥ 70 %**；每次通過套件都要重新生成
兩支存證影片（機載 FPV＋模擬器上帝視角）進 repo。先紅、後綠、留證。

## 第一天：全紅

四個選手各飛 n = 30：

| 選手 | 整合成功率 |
|---|---|
| 特權逐關專家接力（「天花板」） | 13.3 % |
| 五技能通才（KL 錨定微調） | 13.3 % |
| 移動縫 RL 冠軍 | 13.3 % |
| 蛇行行為克隆 | **33.3 %** |

頭條是天花板自己的數字。接力選手直接讀賽道配置、逐關換上該關的
本場專家——特權資訊能買到的最強組合。如果各關以專家原生成功率
（約 0.90）獨立成立，它該考出約 0.73；它考出 **0.133**。

差額就是**接縫稅**。單元測試從乾淨懸停起飛；賽道第二關的起點，
是第一關留下的殘速與側偏。以「活著抵達」為條件的逐關成功率約
0.51——半路進場讓每個 RL 專家的成功率幾乎砍半，因為這種進場狀態
是單元測試從未產生過的條件。單元全綠、整合全紅，而斷點恰好長在
單元測試看不見的地方。

另一個路標：最強選手不是專家接力，而是一個單一的行為克隆，以
2.5 倍勝過特權陣容。它的腳本老師是「從你所在之處轉向可見的縫」，
所以學生繼承了 RL 專家從未訓練過的進場穩健性。**接縫穩健性是被
蒸餾出來的，不是被訓練出來的**——這個觀察決定了下面整條路。

## 爬升：五套陣容，每套都從上一套的失敗直方圖長出來

套件會回報各關型的條件成功率，每次失敗都有名有姓。以下每套陣容
只改一件事、事先註冊，下一套從上一套的直方圖設計。

**1. 賽道微調——0.48，兇手現身。** 把五技能克隆放到隨機賽道本身
上做 PPO 微調（閉環定律：元件要在餵養它的迴路裡訓練），接縫幾乎
全修好：門 1.000、靜態縫 1.000、移動縫 0.848。但蛇行掉到
**0.500**——占全部死因的一半。這是微調安全法則的賽道版：微調修得
好獎勵看得見的東西，同時侵蝕那個獎勵搜尋學不回來的技能
（四十個決策的長鏈）。

**2. 錨定賽道微調——0.39，外加一堂協議課。** 對克隆加 KL 錨，
本想保鏈修縫。只成功一半——蛇行 +13 點、門 −15 點：修復與侵蝕的
交換，在同一張直方圖裡看得見。它還用痛的方式教了我們一課：這套
陣容 n = 30 的畢業考讀 0.567，n = 100 讀 0.390。一個友善的首批
種子推銷了一套平庸陣容。**畢業考從此以合併 n ≥ 60 判**——考試也
要考監考官。

**3. 飛行計畫混編——0.55。** 一張網塞不下五個賽道級技能，就別再
逼它。任務計畫本來就知道賽道組成（就像它知道航點一樣），所以可以
把每一關交給不同的工件：賽道微調通才飛四種關型（條件成功率
0.86–1.00），蛇行克隆接蛇行關。組合勝過死記，+16 點——但蛇行讀
0.579，對照它本場的 0.967：接縫稅在僅剩的關型上第二次被量到。

**4. 熱進場專家——0.62。** 那就到真實接縫上收資料：讓老師接力飛
完整賽道、只留蛇行段、且只取「該段有飛過」的片段（這個過濾器
很關鍵——先前一鍋沒過濾的資料把墜機也默默教了進去）。熱片段克隆
把賽道內蛇行條件成功率抬到 0.645。更好——仍不夠。

**5. 大鍋——0.72，門開。** 剩下的缺口不是覆蓋，是**保真度**。
三柱鏈每關約 13 個連續決策；每步 0.93 的忠實度，鏈式乘積把你封
在我們卡住的位置附近。一次資料加倍——43,488 筆示範、原生鏈形
示範加熱進場片段、驗證準確率 0.963——把專家推過算術要求的門檻。
套件讀數 **72/100，門開了。**

如今 `docs/media/` 裡的存證影片來自通過種子 110004：蛇行 → 開門
→ 移動縫，三關全過、零墜機，FPV 與上帝視角並排。

## 這場考試買到了什麼

三條定律，每條在爬升途中至少被量到兩次：

- **接縫稅。** 組合不是串接；半路進場讓只見過乾淨起飛的專家砍半。
  先為接縫定價，再為技能定價。
- **保真度會複利。** 一條賽道是約四十個決策的連乘；每步 0.93 對
  0.96 聽起來差 3 點，實際是「卡死」與「通過」的差。槓桿是資料量，
  不是架構。
- **微調是把有後座力的手術刀**——修它的食譜、蝕獎勵學不回的東西；
  錨是交換不是免除。通過的陣容用**組合**工件而非融合工件，繞開了
  這個兩難。

還有一條對稱的誠實反證：我們花了五個戰役想讓無人機從廉價機載
觀測*辨識*眼前是哪種障礙，好讓調度用學的而不是用排的。五次全敗；
懸停冷啟動的世界辨識在本 repo 記為未解問題。通過的陣容不做辨識
——飛行計畫本來就知道賽道，如同它知道航點。這個裁定（計畫鍵控的
調度屬佈署合法）是混編陣容能算「可佈署的答案」而非「模擬器戲法」
的原因。

門檻現在是常設考卷，不是一次性表演：新技能先過單元戰役才入池、
任何新組合機制都要面對同樣的 100 條隨機賽道。實機工作已由這道門
解鎖——全目錄第一天考砸的那場考試，現在由這位飛行員在鏡頭前通過。
